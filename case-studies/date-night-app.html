The Concept

Most dating apps optimize for swiping. Date Night flips the model: instead of matching on profiles, users create real date ideas — "Coffee and sunset walk at Dow's Lake, Saturday 6PM" — and others RSVP. The date is the starting point, not the profile.

I designed and developed every piece: user research, UX flows, UI design, React Native frontend, Supabase backend, authentication, database schema, Google Places integration, and the App Store submission.

What I Built
Onboarding flow

Onboarding & auth

Email-based authentication with Supabase Auth. Progressive onboarding that collects name, photo, and date preferences before showing the feed. Password strength validation, email verification, and rate limiting — security built in from the start, not bolted on later.

Date creation

Users describe a date idea with a title, description, location (via Google Places autocomplete), date/time, and optional photo. The creation flow is intentionally simple — three screens, not a form. The goal is to lower the barrier to posting so the feed stays active.

RSVP + chat

RSVP & messaging

When someone RSVPs, both users are connected and can message within the app. Messages are real-time via Supabase Realtime subscriptions. The conversation is tied to a specific date, keeping context clear and reducing the "so what should we do?" problem of other apps.

Challenges

Payments

Discovered mid-development that Stripe doesn't support dating apps. Pivoted the monetization strategy: launched as free with plans for in-app purchases via Apple's native payment system instead of external payment processing. The constraint forced a simpler, better business model.

Dependency hell

React Native + Expo + Supabase + Google Places = a lot of packages with conflicting version requirements. Spent significant time resolving dependency conflicts and build issues. Learned to pin versions aggressively and test on real devices early and often.

Database architecture

Designed the Supabase schema from scratch: users, dates, RSVPs, messages, and reports. Row-level security policies ensure users only see what they should. Wrote migration scripts for every schema change, treating the database like code.

Solo everything

This project was a proof of concept that a designer can ship a real product end-to-end. No engineers, no PM, no QA. Product thinking, visual design, frontend development, backend architecture, and launch — one person, start to finish.

Takeaway

Date Night proved something I'd always believed: designers who can code don't just make better mockups — they make better products. Understanding the technical constraints from the inside changed every design decision I made. And shipping the whole thing — from Figma to the App Store — gave me a level of product intuition that you can't get from handoff-only work.
